# Code of Conduct
## Markdown 写作规范
规范的首要目标是**清晰**，即编辑时的有条理、可维护和渲染效果的美观。

### 原生语法使用
正文中的标题应以名词短语命名；从二级标题开始，且不应跳级。若内容不适宜用标题分割，可采用分割线 `---` 分割，并不开启 TOC 目录。在开启 TOC 目录时，三级标题的分配应均衡使目录美观。分割线不应直接与标题或 Admonition 块相邻。

无序列表应统一使用 `-` 以避免潜在的解析问题。除非列表项达到段落量级，列表项中不应含有句号，应调整表述为一句，分句用分号连接。

粗体一般用于标注专有名词，一对 `**` 之间不应含有前缀与后缀空格。如果专有名词既有中文写法又有英文写法、英文缩写，则只让前者在粗体中。如果没有中文写法则可写为 `**Some Original Phrase (SOP)**` 形式。

斜体只应用于英文书名及一些微妙的表达。

即行内代码 ``` `` ``` 中不应含有 ASCII 及扩展 ASCII 以外的字符。且只应用于代码标识符、命令等，不应用于一般的名称。含中文的引用应使用中文引号或直角引号「」。

`<kbd>` 标签只用于标注按键，如 `<kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>P</kbd>`。

对站内文章的链接，应使用锚点 `@/posts/xxx.md#section`，因为会自带有效性检查。

代码块必须使用 fenced code block 并标注语言。其中对于纯文本选取 `txt`；对于命令选取 `sh` 或 `powershell`。一般来说缩进使用 4 个空格而不是制表符。代码块前相邻的句子必须以句号或者冒号结尾。

注释标记 `[^n]` 应在对应的名词或对应句子的句号后，且不应带来额外的空格。

注释一般放在文末，与正文用分割线隔开。引用使用 Chicago Notes-Bibliography 风格。

### 中英文混排
参考[国家新闻出版署-行业标准-中文出版物夹用英文的编辑规范](https://www.nppa.gov.cn/xxgk/fdzdgknr/hybz/202210/t20221004_445147.html)。

一般而言，中文与英文/数字（包括各种形式但本质文本是英文/数字）之间应加入空格。中文句应尽量避免以英文/数字结尾。

英文的专有名词若存在约定首字母大写写法，则保持首字母大写，如“Windows 系统”；保留“KaTeX”等原始大小写方案。允许使用规范的非英文字母，如弯撇号“L’Hôpital”、连字符“Gauss–Bonnet”、其它语言字母“Möbius”等。

### 公式
如果文中没有公式，或只有数量极少可用 Unicode 代替的公式，则可以不开启公式。

#### 公式嵌入格式
行内公式不应使得字太过狭窄，如行内的分数 `$\frac{x}{y}$` 在恰当情况下应转为 `$x / y$` 形式。行内公式的混排行为与英文相同，特别地，可接受 `$p$-` 加中文这样的用法。

复杂的公式一律使用展示模式。此时与代码块类似，前相邻的句子必须以句号或者冒号结尾。注意尽量让展示公式不使句子从中间断开。

有内容相关数学含义的数字应套上一层 `$$`。引用之前公式的 tag 时则直接在正文写 (1.1) 即可。

#### 命令选择
应注意语义。另外，由于存在反斜杠解析问题（会先进行一次转义再运行 KaTeX），应使用 `\cr` 作为换行；使用 `\\{a_n\\}` 表示数列（作为集合含义时用 `\set{ | }`）。

对数域，使用 `\N \Z \mathbb{Q} \R \Complex`。

恰当使用正体（用 `\mathrm`，在数学性不强时使用 `\text`），如果正体函数与参数之间需要空隙应用 `\operatorname`。

几乎总是应该使用 `\varepsilon` 而不是 `\epsilon`，除非需要特别区分（如 Levi‑Civita 符号）。

省略号使用形如 `a_1, \dots, a_n` 与 `A_1 \oplus \cdots \oplus A_n`。

其它惯例，如极限变量使用 `\varepsilon`，矩阵转置使用 `^\top`。又如：

```math
\begin{aligned}
f: A & \to B \cr
    x & \mapsto g(x)
\end{aligned}
```

### Admonition
一些需要明确语义的 type 如下：
- `tip` 表示提示与技巧；暂时额外用于表示算法
- `info` 表示背景补充与上下文信息；暂时额外用于表示公理
- `note` 表示补充信息，对标数学讲义的 Remark
